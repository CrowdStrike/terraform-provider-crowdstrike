# crowdstrike_sensor_update_policies (Data Source)

This data source provides information about sensor update policies in Falcon.

## API Scopes

The following API scopes are required:

- Sensor update policies | Read


## Example Usage

```terraform
terraform {
  required_providers {
    crowdstrike = {
      source = "registry.terraform.io/crowdstrike/crowdstrike"
    }
  }
}

provider "crowdstrike" {
  cloud = "us-2"
}

# Get all sensor update policies
data "crowdstrike_sensor_update_policies" "all" {}

# Get only enabled policies
data "crowdstrike_sensor_update_policies" "enabled" {
  enabled = true
}

# Get policies for a specific platform
data "crowdstrike_sensor_update_policies" "windows_policies" {
  platform = "Windows"
}

# Get policies with multiple filters
data "crowdstrike_sensor_update_policies" "enabled_windows" {
  platform = "Windows"
  enabled  = true
}

# Get policies using FQL filter
data "crowdstrike_sensor_update_policies" "filtered" {
  filter = "enabled:true+platform_name:'Linux'"
}

# Get policies sorted by name
data "crowdstrike_sensor_update_policies" "sorted" {
  sort = "name.asc"
}

# Get specific policies by ID
data "crowdstrike_sensor_update_policies" "specific" {
  ids = [
    "policy-id-1",
    "policy-id-2"
  ]
}

# Filter by name pattern
data "crowdstrike_sensor_update_policies" "by_name" {
  name = "production"
}

# Filter by description
data "crowdstrike_sensor_update_policies" "by_description" {
  description = "critical"
}

# Output examples
output "all_policies_count" {
  value = length(data.crowdstrike_sensor_update_policies.all.policies)
}

output "enabled_policies" {
  value = data.crowdstrike_sensor_update_policies.enabled.policies
}

output "windows_policy_names" {
  value = [for policy in data.crowdstrike_sensor_update_policies.windows_policies.policies : policy.name]
}

output "first_policy_details" {
  value = length(data.crowdstrike_sensor_update_policies.all.policies) > 0 ? {
    id           = data.crowdstrike_sensor_update_policies.all.policies[0].id
    name         = data.crowdstrike_sensor_update_policies.all.policies[0].name
    description  = data.crowdstrike_sensor_update_policies.all.policies[0].description
    enabled      = data.crowdstrike_sensor_update_policies.all.policies[0].enabled
    platform     = data.crowdstrike_sensor_update_policies.all.policies[0].platform_name
    created_by   = data.crowdstrike_sensor_update_policies.all.policies[0].created_by
    groups_count = length(data.crowdstrike_sensor_update_policies.all.policies[0].groups)
  } : null
}
```

### Optional

- `description` (String) Filter policies by description. Partial matches are supported, but only single words work reliably. Use single words without spaces (e.g., 'malware' works, but 'malware protection' may not return results). For complex description searches with spaces, use the 'filter' attribute instead. Cannot be used together with 'filter' or 'ids'.
- `enabled` (Boolean) Filter policies by enabled status. Cannot be used together with 'filter' or 'ids'.
- `filter` (String) FQL filter to apply to the sensor update policies query. When specified, only policies matching the filter will be returned. Cannot be used together with 'ids' or other filter attributes. Example: `enabled:true`
- `ids` (List of String) List of sensor update policy IDs to retrieve. When specified, only policies with matching IDs will be returned. Cannot be used together with 'filter' or other filter attributes.

Cannot be used together with 'filter' or 'ids'.
